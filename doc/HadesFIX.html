HadesFIX Engine

HadesFIX is a free FIX implementation of the FIX protocol (see FIX Protocol Site) distributed together with a very relaxed license.

Some of the outstanding features of the engine are:

  • FIX version support from 4.0 through 5.0SP2 releases.

  • FIX messages validation as per FIX specifications.

  • Full Java object implementation of the FIX messages. The model library HadesFIXM.jar is a separate component and has no dependencies on other libraries apart of JAXB implementation if FIXML generation is required.

  • Very good performance - latest benchmark counted ~22,000 messages/second on a single Intel Xenon X3440 2.53 GHz processor.

  • All model classes attributes (message tags) are annotated with their FIX version in which have been introduced making programmers life much easier.

  • No FIX message and components/groups description document is required as the object messages are the representation of the message itself.

  • FIXML support starting with FIX version 4.4. FIXML schemas are included and no configuration is necessary to produce or decode a FIXML message. Schema validation can be turned on/off using a configuration flag.

  • Extendable Java object model allows easy addition of custom messages and provide embedded FIXML support using annotations.

  • Flexible architecture allowing multiple complex FIX session scenarios. Each session can be configured separately.

  • Support for FIX message routing (based on DeliverTo and OnBehalfOf fields).

  • Easy to install and configure engine using one configuration XML file per instance.

  • Integrated scheduler component.

  • Easy customizable engine using cascading message Handlers implementing a simple interface.

  • Engine management console command utility using JMX.

  • Enterprise needed security features.
